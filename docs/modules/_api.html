Module _api

BZCC LUA Extended API.

This API creates a full OOP wrapper and replacement the mission functions with an event based system for easier expansion.

Dependencies: _hook

Info:

  • Author: John "Nielk1" Klein

Called before a game object is deleted. Handle is the game object to be deleted. This function will get a lot of traffic so it should not do too much work. Note: This is called after the object is largely removed from the game, so most Get functions won't return a valid value.

PreSnipe
Called when an object is Sniped. Occurs just before the snipe, and can be used to prevent it from happening. This function is called when a SniperShell Ordnance hits an Object and would successfully trigger a Snipe.

Call method: CallAllPassReturn

Return PreSnipeReturnValues

Parameters:

  • curWorld int Current MP World. If you do anything that effects the world, ensure curWorld is 0 to avoid MP Resyncs
  • shooterHandle GameObject The Shooter
  • victimHandle GameObject The object hit
  • ordnanceTeam int Team number of the Ordnance, if applicable
  • ordnanceODF string ODF file of the Ordnance that hit the VictimHandle
  • priorResult HookResult prior event handler's result (optional)

GetNextRandomVehicleODF
Used to select the team's next respawn ODF. Note: Mission scripts tend to handle respawn logic themselves, making this superfluous. Additionally, this function isn't even hooked up to use a value returned from lua, it just calls GetPlayerODF(team) after calling the Lua side as a no-return function.

Call method: CallAllNoReturn

Parameters:

  • team int Team number specified for filtering

    _hook.Add
