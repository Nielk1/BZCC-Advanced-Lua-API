Module _api

BZCC LUA Extended API.

This API creates a full OOP wrapper and replacement the mission functions with an event based system for easier expansion.

Dependencies: `_hook`

Info:

  • Author: John "Nielk1" Klein

PostLoad ()
You probably don't need to implement this
InitialSetup ()
Called before the mission starts up. Preloading assets should be done here.
Start ()
Called when the mission starts for the first time. Use this function to perform any one-time script initialization.
AddObject (h)
Called after any game object is created. Handle is the game object that was created. This function will get a lot of traffic so it should not do too much work. Note that many game object functions may not work properly here.

Parameters:

  • h
DeleteObject (h)
Called before a game object is deleted. Handle is the game object to be deleted. This function will get a lot of traffic so it should not do too much work. Note: This is called after the object is largely removed from the game, so most Get functions won't return a valid value.

Parameters:

  • h
Update ()
Called once per tick after updating the network system and before simulating game objects. This function performs most of the mission script's game logic.

  • killersHandle handle Handle for the object that killed this object, if present
PreOrdnanceHit (shooterHandle, victimHandle, ordnanceTeam, ordnanceODF)
Called when an ordnance hits an object. This technically happens just before any damage is applied.

Parameters:

  • shooterHandle handle Handle of the Shooter
  • victimHandle handle Handle of the object hit
  • ordnanceTeam int Team number of the Ordnance, if applicable
  • ordnanceODF string ODF file of the Ordnance that hit the VictimHandle
PreSnipe (curWorld, shooterHandle, victimHandle, ordnanceTeam, ordnanceODF)
Called when an object is Sniped. Occurs just before the snipe, and can be used to prevent it from happening. This function is called when a SniperShell Ordnance hits an Object and would successfully trigger a Snipe.

Parameters:

  • curWorld int Current MP World. If you do anything that effects the world, ensure curWorld is 0 to avoid MP Resyncs
  • shooterHandle handle Handle of the Shooter
  • victimHandle handle Handle of the object hit
  • ordnanceTeam int Team number of the Ordnance, if applicable
  • ordnanceODF string ODF file of the Ordnance that hit the VictimHandle

Returns:

    int PreSnipeReturnValues see ScriptUtils.PreSnipeReturnValues
PreGetIn (curWorld, pilotHandle, emptyCraftHandle)
Called when a pilot gets into a ship. Can be used to prevent it.

Parameters:

  • curWorld int The current MP World. If you do anything that effects the world, ensure curWorld is 0 to avoid MP Resyncs
  • pilotHandle handle Handle of the Pilot
  • emptyCraftHandle handle Handle of the Empty craft

Returns:

    int PreGetInReturnValues see ScriptUtils.PreGetInReturnValues
PrePickupPowerup (curWorld, me, powerupHandle)
Called when a powerup is picked up. Can be used to prevent it.

Parameters:

  • curWorld int The current MP World. If you do anything that effects the world, ensure curWorld is 0 to avoid MP Resyncs.
  • me handle Handle of the object picking up the Powerup
  • powerupHandle handle Handle of the powerup object

Returns:

    int PrePickupPowerupReturnValues see ScriptUtils.PrePickupPowerupReturnValues
PostTargetChangeCallback (craft, previousTarget, currentTarget)
Called when the user? changes targets? Can be used to trigger events on a target change?

Parameters:

  • craft handle Handle of the object that just changed Targets
  • previousTarget handle Handle of the previous Target
  • currentTarget handle Handle of the new Target
PostRun ()
Called at the End of the mission, shortly before game returns to the Shell UI.
GetNextRandomVehicleODF (team)
Used to select the team's next respawn ODF. Note: Mission scripts tend to handle respawn logic themselves, making this superfluous.

Parameters:

  • team int Team number specified for filtering

Called when an IFace command is triggered. Use CalcCRC(string) to determine the command from the crc value.

Parameters:

  • crc int CRC of the Command to process
